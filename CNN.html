How to handle Image with neural netowrk

Imagine you are processing an image. Each pixel would have three values (red , green , blue). This means you have to handle \(128 \times 128 \times 3 = 49152 \) real number input when handling 128 x 128 pixel images! Now let's consider you pass though a single layer that transform to 64 features. \[ \vec{x}_{1\times 41952}^T . \bf{M}_{1 \times 64} \] which takes approximate 3,000,000 unit of computation !

This works but we hope to find a better way to handle this problem. Noticing that many image features are coming from its neighbour. For example you will consider the feature is coming from the ear instead of nose if it is just one pixel next to the ear. As a our idea if a particular ("area") patch of image contains a feature instead of particular pixel

Next we have to apply filter to these mini batch. let's say each batch is 3x3x3. Then each batch can apply to few filter as a feature. As a result it will form a 128 x 128 x 4 layer. This doesn't solve our problem of having a huge image

We solve the problem by applying a max pool. What this does is just take max of each 2x2 region.

After max pooling we will get the layer of 64x 64 x 4 . This is called downsampling. Note that this procedure continues and the filter may form different deepth and then do some different kinds of pooling but the idea is still lower the size of the input.

So what max pool / other pool is just extra the most important information from the image (?)
